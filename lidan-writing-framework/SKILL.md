# 李诞七步写作框架

这个 skill 用于把复杂概念写成“读者看完不仅知道是什么，而且能讲给别人听”的中文文章。适合知识科普、商业解释、AI/技术概念、语言学、心理学、社会学等主题。

## 触发方式

当用户使用下面任一表达时，应用本框架：

- `$lidan-writing-framework`
- `/写作框架`
- `/七步写作`
- `/李诞写作`
- “用李诞七步框架写”
- “把这个概念讲得深入浅出”
- “写一篇能让普通人看懂的科普文章”

## 工作方式

先确认用户要解释的核心概念、目标读者和大致长度。如果信息不足，做合理假设并在开头简短说明。

输出文章时，默认采用七步结构。除非用户要求展示框架，否则不要机械暴露每一步的小标题；可以把结构隐藏在正文里，让文章自然流动。

## 七步框架

### 1. 开场故事

不要先下定义。用一个具体场景开场，比如美术馆、咖啡馆、会议室、实验室、地铁、饭桌。让两个人围绕一个具体问题发生争论，自然引出核心概念。

要点：

- 场景要具体，有画面。
- 用对话或冲突引出问题。
- 制造一点悬念，让读者愿意继续看。

### 2. 三种错误答案

先带读者走三条常见弯路。每个错误答案都要像真的，有吸引力，然后用反例或常识击穿它。

结构：

- 错误观点 A + 为什么诱人 + 致命反例
- 错误观点 B + 为什么诱人 + 致命反例
- 错误观点 C + 为什么诱人 + 致命反例

写法要干脆，不要温吞。这里的作用是清理读者脑子里原有的误解。

### 3. 正确答案

用一句人话说清概念本质。不要堆术语，不要假装深奥。

结构：

- 一句话定义
- 为什么是这样
- 一个具体例子
- 一个可验证标准：读者以后怎么判断自己是否理解对了

### 4. 触类旁通

把核心概念放到 3 到 5 个不同领域里验证。每个领域都要有具体例子，帮助读者建立“这个概念不是孤例，而是一种通用结构”的感觉。

可选领域：

- 商业
- 心理学
- AI/技术
- 日常生活
- 历史
- 教育
- 组织管理

这一部分是文章的核心，要让读者产生“原来很多事都是一回事”的感觉。

### 5. 对比制造冲击

找一个强对比来放大理解效果，比如过去 vs 现在，东方 vs 西方，人 vs AI，少量 vs 大量，直觉 vs 数据。

对比不是为了炫技，而是为了让读者看到差异为什么重要。

### 6. 结尾升华

回到开场故事，形成闭环。给出更大的洞察，但不要喊口号，不要强行鸡汤。

好的结尾应该做到：

- 呼应开头。
- 让读者觉得自己理解了一个更大的东西。
- 留一点开放空间，而不是把话说死。

### 7. 延伸阅读

给出 3 到 5 个延伸方向，可以是书、概念、关键词、研究方向或现实案例。每个方向用一句话说明为什么值得看。

## 输出要求

- 默认使用中文。
- 写得像人在解释，不像百科词条。
- 尽量使用短句和具体场景。
- 少用“底层逻辑、赋能、范式、生态、抓手”等空泛词。
- 如果主题很专业，先降维解释，再逐步加深。
- 如果用户要求成稿，直接给完整文章。
- 如果用户要求大纲，输出七步大纲和每一步写作要点。
- 如果用户给了草稿，按七步框架诊断哪里缺故事、哪里误解清理不够、哪里没有触类旁通。

## 质量自检

交付前检查：

- 开头有没有具体场景，而不是抽象定义？
- 错误答案是否足够像真问题？
- 正确答案能否用一句话讲清？
- 有没有跨领域例子，而不是只在一个领域里绕？
- 对比是否带来认知冲击？
- 结尾是否回到了开头？
- 读者看完能不能讲给别人听？
