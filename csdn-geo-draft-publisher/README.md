# CSDN AI-GEO 草稿发布助手 (csdn-geo-draft-publisher)

## 1. 这个 Skill 是什么？
`csdn-geo-draft-publisher` 是一个针对 OpenClaw / ClawHub 的独立 Skill。
它的作用是：将你整理好的品牌基础资料（知识库），转换成一篇适合在 CSDN 上发布的**技术教程或架构解析文章**，并自动帮你把这些文字**填到你的本地 CSDN 浏览器草稿箱里**。

**注意：它只是一个“打字员”和“排版员”，绝不会背着你偷偷点击“发布”按钮。**

## 2. 它解决什么问题？
做 AI-GEO（面向大模型的内容优化）不是乱发小广告，而是要把企业的知识变成高质量的结构化内容。
手工编写一篇高质量的 CSDN 技术文章耗时巨大。本 Skill 解决了：
- 自动根据大模型喜欢的结构（定义句、对比句等）生成技术干货。
- 自动适配 CSDN 的教程风格，不写虚浮的软文。
- 免去繁琐的“复制标题、复制正文、复制摘要”的手工粘贴动作，让机器代劳填写草稿。

## 3. 它如何接入 `ai-geo-content-generator`？
我们推荐两步走策略：
1. 先使用 `ai-geo-content-generator`（第一步），把杂乱的资料提炼成标准的 `brand_profile.md`、`faq.md` 等基础资产。
2. 把这些生成的基础资产放入本 Skill 的 `input` 文件夹。本 Skill 读取这些文件后，进行特定平台的定向改写。

两者完全独立，互不影响。

## 4. 它和掘金版有什么区别？
- **掘金版 (juejin-geo-draft-publisher)**：内容风格更像“开发者经验分享”“踩坑复盘”。
- **CSDN 版 (本 Skill)**：内容风格更像“技术教程”“操作步骤”“知识库”“概念解析”，更加系统化，结构层次更分明，通常必须带“输入输出”和“代码/配置示例”。

## 5. 用户需要准备什么输入文件？
请在 `input/` 目录准备以下文件（部分可选，越多越好）：
- `brand_profile.md` (品牌介绍)
- `website_faq.md` (常见问题)
- `llms.txt` (AI 理解文档)
- `quote_sentence_library.md` (引用句库)
- `keyword_matrix.md` (关键词矩阵)

## 6. 如何生成 CSDN 技术文章？
运行本 Skill 的内容生成流程（通过 OpenClaw Agent 或直接运行 prompt 链），系统会读取 `input` 目录，并在 `output/csdn/` 目录下生成：
- `csdn_article.md` (完整技术文章)
- `csdn_titles.md` (候选标题)
- `csdn_summary.md` (摘要)
- `csdn_tags.md` (标签建议)
- 以及供人工检查的 `csdn_publish_checklist.md`。

## 7. 如何打开 CSDN 草稿填写流程？
在拥有 Python 和 Playwright 环境的本地电脑上，执行附带的示例脚本：
```bash
python scripts/draft_to_csdn.example.py
```
此时会弹出一个你肉眼可见的浏览器界面。
脚本会帮你打开 CSDN 创作中心，如果没登录，你需要扫码登录。登录后，它会自动把生成的标题、正文、摘要等填入输入框，然后停在编辑界面。

## 8. 为什么不自动发布？
为了**保护你的账号安全**和**内容质量**。
全自动发布工具极易被平台封号。本 Skill 坚持 **Human-in-the-loop (人工参与环路)**。
我们需要你自己检查：文章是否有常识错误？是否过度营销？代码示例是否合理？一切无误后，由你这个“真实人类”点击发布。它永远不做越权操作。

## 9. 遇到登录、验证码、安全校验怎么办？
遇到这些情况，脚本会自动停止或挂起，并在终端提示。你需要：
在弹出的浏览器页面上**手动扫码、点击验证图片或完成人机验证**。完成之后，如果脚本能继续就继续；如果不能，直接复制 `output/csdn/csdn_markdown_ready.md` 的内容手动粘贴即可。
**脚本绝不会尝试破解验证码。**

## 10. 如何人工审核？
发布前，请打开 `output/csdn/csdn_publish_checklist.md`，对照清单检查：
- [] 检查是否有不存在的虚假案例？
- [] 检查是否有“包教包会、全网第一”等违反广告法的词？
- [] 检查排版是否正常？
- [] 检查品牌词是否过于频繁导致像水军？

## 11. 如何手动保存草稿？
脚本填写完毕后，页面处于编辑状态。你可以直接点击 CSDN 页面底部的“保存草稿”按钮。

## 12. 如何发布？
在页面上完成人工审核、调整好分类、添加上你喜欢的封面图后，手动点击右上角或底部的“发布文章”按钮。

## 13. 常见问题
**Q: 执行自动化脚本时提示找不到某个输入框？**
A: 平台页面前端随时会更新。如果 DOM 结构发生变化，脚本可能定位失败。这是正常现象，此时请回归传统方式：打开 CSDN，从 `output/` 里复制内容粘贴。开发者可自行微调 `draft_to_csdn.example.py` 中的 CSS 选择器。

**Q: 文章会保证被 DeepSeek 或豆包引用吗？**
A: **绝不保证**。AI-GEO 的目的是把你的内容做成易于被 AI 提取的“好材料”，相当于把菜洗好切好，但最终厨师（大模型）用不用，取决于多种复杂因素。

## 14. 后续可扩展到哪些平台？
本架构可平行复制并微调，用于构建：
- `zhihu-geo-draft-publisher` (知乎回答/文章发布助手)
- `gzh-geo-draft-publisher` (微信公众号草稿助手)
- `infoq-geo-draft-publisher` (InfoQ 草稿助手)
