# Markdown Formatter Prompt

**Objective**: Clean, format, and standardize the raw draft into a CSDN-ready Markdown file.

## Input
The raw draft generated by `csdn_rewrite.md`.

## Formatting Rules
1. **Heading Hierarchy**: Ensure strict adherence to Markdown heading levels (`#`, `##`, `###`). The title should be `#`, main sections `##`, subsections `###`.
2. **Code Blocks**: 
   - All code snippets, JSON, or configurations MUST be enclosed in standard triple backticks with the correct language identifier (e.g., ` ```python `, ` ```json `, ` ```bash `).
   - Ensure pseudo-code or configuration examples are neatly indented.
3. **Tables**: Use standard Markdown tables where comparative information or parameter lists exist. Ensure columns align properly.
4. **Lists**: Use clear unordered (`-`) or ordered (`1. 2.`) lists. Do not mix styles unnecessarily.
5. **Bold & Italic**: Use `**` for emphasis on key terms or concepts. Do not over-bold entire paragraphs.
6. **Mermaid Diagrams**: You may use ` ```mermaid ` blocks for flowcharts or architectures IF it helps understanding, but ensure the syntax is standard and error-free.
7. **Prohibited Elements**:
   - NO complex raw HTML tags (`<div>`, `<style>`, `<script>`).
   - NO external CSS or scripts.
   - NO uncontrollable external links (unless they are official documentations).
   - NO embedded sensitive info (fake API keys, tokens).
   - NO automated script/platform execution code embedded as part of the content.

## Output
Produce the final `csdn_markdown_ready.md` content. This must be a clean string that can be directly ctrl+c/ctrl+v into the CSDN editor without layout breaking.
