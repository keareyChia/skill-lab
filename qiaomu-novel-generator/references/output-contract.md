# Output Contract

Use this file before drafting the final story.

## Prewrite Before Draft

For a new story request, do not default to immediate full prose unless the user explicitly asks to skip discussion.

1. If the premise is vague, use `references/prewrite-interview.md` and ask with compact numbered choices.
2. If the premise is usable, first provide `小说如何吸引人` and `大纲`.
3. Draft the full story only after the user confirms the direction, or when the user explicitly says `直接写`, `不用讨论`, or `按默认直接写`.

The prewrite response is part of quality, not delay. It lets the story earn its hook, desire, conflict ladder, payoff, and ending before prose begins.

Before prewriting, select from `references/story-engine-library.md`:

- one primary emotional payoff;
- one high-pressure relationship;
- one visible conflict arena;
- 2-4 core plot engines;
- one escalation ladder;
- one ending hook or aftertaste.

If the user gives any plot idea, named work, or author signal, also select from `references/inspiration-remix-playbook.md`:

- 3-6 classic beat cards to offer the user;
- one recommended combination;
- the craft features to borrow;
- at least three elements to change so the result is a new story.

## Complete Short Story Requirements

- Give the story a real ending. Do not stop at a trailer, synopsis, or chapter-one fragment unless the user asks for a serial opening.
- Default length: 1800-4000 Chinese characters.
- Put an irreversible disturbance within the first 3 paragraphs.
- Keep the first 3 paragraphs physically legible. The reader may wonder `who`, `why`, or `what danger`, but should not be confused about whether a person is alive, dead, dreaming, supernatural, or metaphorical unless that is the intended genre rule.
- Give the protagonist one visible desire and one hidden wound.
- Escalate conflict at least 3 times.
- Use 2-4 plot engines deliberately; do not let a story rely on only generic mood.
- Use concrete scenes. Avoid long explanations of background.
- Use dialogue to force choices, not to recite information.
- Use one recurring image or object and return to it near the end.
- End with resolution plus aftertaste: the core conflict should resolve, but the final image should echo beyond the page.

## Drafting Rules

- Prefer verbs and images over adjective stacks.
- Short sentences can create pressure; longer sentences can slow time. Use both intentionally.
- Do not sacrifice clarity for a clever hook. In the opening, strange phrases need a concrete anchor immediately.
- Do not explain the technique in the visible story.
- Do not put section labels inside the story unless the user asks for a structured artifact.
- If using a source fragment, keep facts and promises but rewrite scene logic when needed.
- If the user asks for a famous plot reference, extract only the underlying structure and create new names, setting, stakes, rules, and resolution.
- If the user asks for a living author's style, translate it into craft features and do not directly imitate the distinctive voice.
- Apply `references/anti-ai-language.md` before returning. In story narration, target zero `不是X，而是Y` style sentences.

## Revision Rules

If the story feels flat, diagnose the most important failure and fix that first:

1. Hook weak: start later, closer to the wound.
2. Desire weak: make the protagonist want something visible now.
3. Conflict weak: close the safe option.
4. Dialogue weak: turn information into accusation, bargain, or test.
5. Image weak: choose one object and make it carry memory or cost.
6. Ending weak: make the final image reframe the first scene.
7. AI-ish language: replace formulaic contrast or teaching-tone summary with action, image, dialogue, and consequence.
8. Opening misread: if a line implies impossible action by accident, rewrite it into literal danger plus one sharp image.
