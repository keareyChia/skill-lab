# Story Engine Library

Use this file during the Intent Hook, Prewrite Interview Hook, Story Strategy Hook, and Plan Hook.

This is a modular engine library, not a list of fixed plots. The goal is to make any story request easier to hold: first choose the reader's emotional payoff, then bind characters under pressure, then select 2-4 engines, then build escalation and hooks.

For famous film/TV/novel inspirations and author-style signals, use `references/inspiration-remix-playbook.md` first. This library then supplies the engines that make the remix work.

## Research Anchors

This library synthesizes broad public craft advice and the user's own short-drama framework:

- Web fiction and serial stories rely on expectation, crisis, payoff, and continued curiosity.
- Strong short-form drama compresses each unit into hook, escalation, and cliffhanger.
- Escalation works best when the worsening problem is character-driven, not only bigger spectacle.
- Chapter endings should feel changed or complete enough to stop, while raising tension or a new question.
- Progression fantasy and cultivation-style stories work because readers track visible growth, rank, resource scarcity, and proof.
- Beat sheets are useful as editing tools, but should not make the story stale or mechanical.
- When the user asks to search a specific reference, use `source-research-remix.md` to turn public context into abstract functions before selecting engines here.

## Universal Containment Formula

When a premise feels hard to write, reduce it to five layers:

```text
情绪承诺 -> 高压关系 -> 冲突场 -> 引擎组合 -> 升级/钩子节奏
```

Expanded:

```text
强设定 + 强委屈/缺口 + 强关系 + 强对手 + 强反转 + 强钩子 = 可追读故事
```

If one layer is weak, strengthen the next layer. For example:

- Setting ordinary? Make the relationship closer.
- Antagonist ordinary? Give them evidence, status, or public leverage.
- Power system thin? Add scarcity, rules, and cost.
- Plot too quiet? Move the conflict into a public arena.
- Ending weak? Return an earlier object with changed meaning.

## Core Plot Engines

Pick 2-4 engines. One is the main engine; the others should create pressure or reversal.

### 1. Hidden Identity

The protagonist is misread as weak, poor, guilty, fake, or irrelevant while holding a real rank, skill, debt, or proof.

Best payoff: public proof changes power.

Avoid: effortless bragging. Seed the hidden ability through objects, habits, old scars, or tiny choices.

### 2. Rebirth Or Second Chance

The protagonist returns with knowledge of betrayal, disaster, or a missed chance.

Best payoff: the same trap now catches the trapper.

Avoid: omniscient convenience. The future should bend when the protagonist acts.

### 3. Contract Binding

Two people are forced together by marriage, debt, mission, business, curse, oath, or survival.

Best payoff: forced interest becomes real choice, or the contract becomes a weapon.

Avoid: static bickering. Every scene should change the terms of the contract.

### 4. Mistaken Identity

A key person recognizes the wrong savior, heir, lover, criminal, or genius.

Best payoff: an object or wound proves the truth when the lie has grown expensive.

Avoid: delaying the reveal only because characters refuse to talk.

### 5. Double-Strong Game

Both sides hide competence and test each other.

Best payoff: one apparent defeat is revealed as bait.

Avoid: making one side stupid to make the other look smart.

### 6. Progression Ladder

The protagonist visibly grows in rank, skill, wealth, authority, or social proof.

Best payoff: the reader sees a before/after gap and knows the cost.

Avoid: abstract leveling. Tie growth to a concrete resource, wound, rule, or trial.

### 7. Public Arena

The conflict happens where reputation can change: auction, wedding, divorce court, family banquet, sect trial, livestream, public audit, school ranking, council hall.

Best payoff: a witness who dismissed the protagonist must react.

Avoid: private revenge for a public humiliation unless the private scene creates a larger public consequence.

### 8. Auction / Appraisal / Black Market

Value is hidden in an object, bidder, rule, or old debt.

Best payoff: the cheap item is priceless, the prized item is poison, or the bid exposes identity.

Avoid: treasure appears randomly. Make the protagonist's knowledge costly or earned.

### 9. Trial / Tournament / Assessment

A rule-bound contest forces proof under observation.

Best payoff: the protagonist wins by understanding the rule better than the stronger opponent.

Avoid: pure power crushing. Add rule exploitation, moral price, or hidden constraint.

### 10. Conspiracy Clue Chain

Each clue answers one question and opens a worse one.

Best payoff: early innocent detail returns as evidence.

Avoid: dropping the culprit from nowhere.

### 11. Sealed Memory / Missing Past

A lost memory, erased record, old photo, token, or wound hides a choice the protagonist made.

Best payoff: the protagonist already paid a price before the story began.

Avoid: memory reveal that only explains, not changes the current choice.

### 12. Forbidden Bargain

A bargain solves the visible problem while creating moral, social, or supernatural debt.

Best payoff: the protagonist wins by paying a price the opponent cannot imagine.

Avoid: free magic or free help.

### 13. Enemy-As-Protector

The apparent enemy has been blocking a worse danger.

Best payoff: hatred flips into debt without removing the harm.

Avoid: using protection to excuse cruelty too cheaply.

### 14. Class / Family Displacement

The protagonist is displaced from the role they should have: heir, daughter, founder, disciple, owner, original creator.

Best payoff: legitimacy returns through action and evidence.

Avoid: only identity reveal. The identity should solve one problem and create a larger one.

### 15. System / Rule Exploit

A system, game, law, school rule, platform rule, AI rule, cultivation rule, or market rule is exploitable.

Best payoff: the protagonist notices a loophole everyone else ignores.

Avoid: rule dump. Show the rule by trapping someone in it.

### 16. Outsider Ritual Barrier

The protagonist has real competence but does not yet know the local ritual: etiquette, jargon, ranking rules, hearing procedure, guild protocol, sect taboo, court custom, or institutional paperwork.

Best payoff: they learn the surface ritual, then use deeper competence, evidence, or insight to beat the insiders.

Avoid: framing unfamiliarity as stupidity. The story should distinguish `not knowing this circle's rules yet` from `being weak`.

### 17. Object Proof

A scar, ring, ledger, old coin, jade, broken phone, recipe, code comment, contract clause, or train ticket carries the truth.

Best payoff: the object changes meaning in the last scene.

Avoid: object only as decoration.

### 18. Forced Low Point

The protagonist's first win causes a larger enemy, public misunderstanding, or worse loss.

Best payoff: delayed release. Let the reader feel pressure before the comeback.

Avoid: endless憋屈. Give small proofs and partial wins along the way.

### 19. Backstage Mission POV

A famous-looking or public-facing mission is told through the person who coordinates logistics, absorbs blame, and understands the hidden joints of the system.

Best payoff: the backstage worker turns invisible labor into public proof.

Avoid: making the story only a complaint about process. The protagonist needs a desire, a moral line, and a choice.

### 20. Stakeholder Goal Collision

Multiple parties claim to support the same project, but each wants a different private result: credit, delay, scapegoat, promotion, budget, revenge, or silence.

Best payoff: the protagonist forces private goals into the open.

Avoid: vague "office politics". Give every faction a concrete lever and fear.

### 21. Procedure-As-Weapon

Rules, documents, logs, approvals, budgets, rosters, contracts, or meeting notes are first obstacles, then become evidence or traps.

Best payoff: a boring artifact changes status in a public arena.

Avoid: process manual exposition. Show the rule by making it hurt someone in scene.

### 22. Scapegoat Project

A project is designed so that success enriches others and failure lands on the protagonist.

Best payoff: the protagonist changes the definition of success, or proves the failure was pre-arranged.

Avoid: the protagonist merely being lucky. The reversal must come from evidence, timing, and a risky choice.

## Escalation Ladders

Use one ladder as the spine.

### Face-Slapping Ladder

```text
被低估 -> 被公开羞辱 -> 对手拿出证据/权力 -> 主角先忍或试探 -> 小反击 -> 对手升级陷害 -> 主角亮出第一层底牌 -> 更强敌人出现 -> 公开翻盘 -> 留下更大的身份/阴谋钩子
```

### Revenge Ladder

```text
旧伤/背叛 -> 机会重来或新证据 -> 主角布小局 -> 反派以为得手 -> 第一次反杀 -> 牵出保护伞 -> 主角失去一样东西 -> 终局让反派亲手承认或自毁
```

### Suspense Ladder

```text
异常事件 -> 第一个合理解释 -> 细节不对 -> 第二个更可怕解释 -> 证人/证物反转 -> 主角发现自己也在局中 -> 真相解决当前案子 -> 最后一件物品打开更大谜团
```

### Romance Pull Ladder

```text
误会/契约 -> 被迫相处 -> 一次保护被误读 -> 尊严冲突 -> 关系升温 -> 旧人/旧案回来 -> 一方做出代价选择 -> 迟来的真相 -> 公开选择
```

### Progression Ladder

```text
低位/废柴标签 -> 小资源 -> 规则/训练/试炼 -> 第一次证明 -> 代价暴露 -> 更高阶压制 -> 规则突破 -> 公开认可 -> 新等级带来新敌人
```

### Organization Infighting Ladder

```text
重大任务落下 -> 主角接到模糊授权 -> 各方表面配合 -> 第一份文件/流程暴露矛盾 -> 主角小修补反被推成责任人 -> 对手抢功或甩锅 -> 主角找到程序物证 -> 更高层试图压下 -> 公开场合反转定义责任 -> 主角赢下当前局但看见更大的系统问题
```

## Chapter Or Episode Hook Bank

A hook does not always mean a cliffhanger. It can be a changed situation plus an urgent question.

Use one of these at the end of a chapter, scene, or short-story section:

- Disaster: the safe plan fails.
- Arrival: the wrong person enters.
- Proof: an object proves a forbidden truth.
- Reversal: the small win creates a worse problem.
- Choice: the protagonist must pick between two costs.
- Exposure: a private secret becomes public.
- Delay: the answer is interrupted by a more urgent danger.
- Misread: a witness draws the wrong conclusion.
- Betrayal: helper is compromised.
- Countdown: the deadline moves earlier.
- Price: the solution demands a sacrifice.
- Recognition: someone finally sees the protagonist's value, but too late.

Avoid fake hooks. A hook should change what the next scene must handle.

## Short Story Adaptation

For a complete short story, do not copy short-drama episode logic mechanically. Compress it:

```text
Opening hook -> pressure relationship -> 2-3 escalations -> one strong payoff -> final aftertaste
```

For a serial outline, use:

```text
first 3 chapters/episodes: core wound + first proof + first big reversal
every 3-5 units: small climax
every 10-15 units: major reveal, relationship flip, or new enemy
final stretch: all clues converge, payoff current promise, open next-level promise if needed
```

## Prewrite Selection Rule

When the user asks for a story, choose defaults from this library:

1. Select primary emotional payoff.
2. Select one high-pressure relationship.
3. Select one visible conflict arena.
4. Select 2-4 core plot engines.
5. Select one escalation ladder.
6. Select one ending aftertaste.
7. If the user provided a classic work or author signal, add 2-4 remix beat cards from `inspiration-remix-playbook.md`.

Then present compact choices using `references/prewrite-interview.md`.

## Anti-Formula Rule

These are engines, not excuses for lazy cliché.

- Every familiar trope needs one specific fresh object, profession, rule, wound, or cost.
- Do not solve everything through identity reveal.
- Do not make antagonists stupid; make them certain for a reason.
- Do not stack too many engines. If the story has no center, return to the emotional payoff.
- The protagonist should win by choice, skill, evidence, sacrifice, or insight, not by author convenience.
